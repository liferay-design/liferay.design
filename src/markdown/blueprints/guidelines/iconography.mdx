## Introduction

Visually consistent illustrative icons help create a unified experience across the many Liferay sites, services, and assets. Illustrative iconography helps communicate ideas, concepts, feelings, etc. Their role is to support copy, while not relying on the copy to be meaningful and communicative. This means icons are straightforward and easily interpreted – a well designed icon has a single focal point and is “readable” without the use of words.

## Guidelines

The guidelines provided here are meant to enable the creation of icons that are visually and functionally consistent with the Liferay brand. Although these rules should be followed as closely as possible, there are cases where making an exception will improve an icon, rather than detract from it. Use your own best judgment to determine if an exception is necessary for the betterment of an icon.

## Icon Anatomy

<br />

Elements of an icon in order that they are layered:

1. **Artboard** – The artboard is the area in which an icon is constructed. Artboards are always square. All icon elements should be contained in the artboard and not be cut off.

2. **Background** – The background is solid white and fills the entirety of the icon. A solid background ensures that the icon can be used on any color of background and maintain its visual integrity

3. **Shade** – While all icons are flat perspective, the shade provides depth, texture and adds color.

4. **Structure** – The structure provides context to the focus of an icon. While the Focus should communicate the core idea the structure gives context and additional meaning to the focus.

5. **Focus** – The focus is the primary element that communicates and idea within an icon. The focus can often stand on it's own as an idea but it is okay if the focus relies on the structure to provide the full idea of an icon.

## General Rules

<br />

1. An icon has a **flat perspective** (2D)

<DoDont 
    good="/images/blueprints/iconography/do-dont/perspective-do.png"
    bad="/images/blueprints/iconography/do-dont/perspective-dont.png"
/>
<br/>
<br/>
<br/>

2. An icon has no more than **2 layered elements**.

<DoDont 
    good="/images/blueprints/iconography/do-dont/layers-do.png"
    bad="/images/blueprints/iconography/do-dont/layers-dont.png"
/>
<br/>
<br/>
<br/>

3. An icon **does not use letters or words**. An effective icon is language and culture neutral, with the exception of a currency symbol.

<DoDont 
    good="/images/blueprints/iconography/do-dont/letters-do.png"
    bad="/images/blueprints/iconography/do-dont/letters-dont.png"
/>
<br/>
<br/>
<br/>

4. An icon communicates a **single idea**. Complex ideas are either split into multiple icons or a single icon is used to focuses on the overarching idea.

<DoDont
	good="/images/blueprints/iconography/do-dont/single-idea-do.png"
	bad="/images/blueprints/iconography/do-dont/single-idea-dont.png"
/>

## Element Sizes

<br />

1. **Artboard** – The artboard is the area of an icon. Every icon artboard is a 32px square with no content overflow. After creating an icon the artboard is scaled larger or smaller depending on the need.

<DoDont 
    good="/images/blueprints/iconography/do-dont/artboard-size-do.png"
    bad="/images/blueprints/iconography/do-dont/artboard-size-dont.png"
/>
<br/>
<br/>
<br/>

2. **Focus** – The focus is the boldy colored point of an icon that “speaks”. Each icon has a focal point with:
    <br />

a. A max width or height of 16px

<DoDont 
    good="/images/blueprints/iconography/do-dont/focus-size-do.png"
    bad="/images/blueprints/iconography/do-dont/focus-size-dont.png"
/>
<br/>
<br/>
<br/>

b. A total area not exceeding 200sq pixels

<DoDont 
    good="/images/blueprints/iconography/do-dont/focus-area-do.png"
    bad="/images/blueprints/iconography/do-dont/focus-area-dont.png"
/>
<br/>
<br/>
<br/>

c. A stroke width of 2px (if applicable)

<DoDont 
    good="/images/blueprints/iconography/do-dont/focus-stroke-do.png"
    bad="/images/blueprints/iconography/do-dont/focus-stroke-dont.png"
/>
<br/>
<br/>
<br/>

3. **Shade** – The shade gives depth and texture to an icon. It is a triangular shape filling the lower right portion of each element.
    <br />

a. It **fills 50% or less** of the totall area of an element

<DoDont 
    good="/images/blueprints/iconography/do-dont/shade-area-do.png"
    bad="/images/blueprints/iconography/do-dont/shade-area-dont.png"
/>
<br/>
<br/>
<br/>

b. The angled side is a **45° angle** by default

<DoDont 
    good="/images/blueprints/iconography/do-dont/shade-angle-do.png"
    bad="/images/blueprints/iconography/do-dont/shade-angle-dont.png"
/>
<br/>
<br/>
<br/>

4. **Structure** – The structure consists of **1px strokes** and on rare occasions solid shapes. On standard shaped icons (squares, circles, rectanges, ect) a **3px minimum padding between the outer stroke and the focus** is prefered.

![Structure Size Specs](/images/blueprints/iconography/do-dont/structure-size-specs.png)

<br/>
<br/>
<br/>

5. **Background** – The background is solid white and **fills the entirety of the icon**. A solid background ensures that the icon can be used on any color of background and maintain its visual integrity.

![Background Size Specs](/images/blueprints/iconography/do-dont/background-size-specs.png)

## Stroke

<br />

1. **Focus** – The stroke of a focus element has:
    <br />

-   A 2px stroke

-   A round cap

-   1px rounded external corders when possible

-   0.25px minimum external corners

![Focus Stroke Specs](/images/blueprints/iconography/do-dont/focus-stroke-specs.png)

<br/>
<br/>
<br/>
<br/>
<br/>

2. **Structure** – The stroke of the structure has:
    <br />

-   A 2px stroke
-   A round cap
-   2px round external corners when possible
-   0.25px minimum external corners

![Structure Stroke Specs](/images/blueprints/iconography/do-dont/structure-stroke-specs.png)
